Why it exists

Documentation generation with an audit trail.

Docs Forge builds a persistent .docs-forge/ knowledge base before it writes the final docs. The output is resumable, reviewable, and grounded in file citations instead of loose summaries.

It is intended for architecture docs, API references, runbooks, product walkthroughs, tutorials, and docs-site setup across common documentation frameworks. The same workflow is packaged for Codex, Claude Code, Antigravity, and general AGENTS.md readers.

01

Reads the repo first

Scans the entire codebase, then prioritizes manifests, entrypoints, routes, public exports, configs, tests, and existing docs before writing new material.

02

Asks only for intent

Separates what code can prove from what only a maintainer can answer: audience, naming, supported APIs, deprecation, tone, and use cases.

03

Targets real frameworks

Outputs pages and navigation for Fumadocs, Docusaurus, Mintlify, Nextra, Starlight, and MkDocs conventions.

Agent support

One workflow, multiple entrypoints.

Codex

Install as a Git-backed marketplace plugin and invoke $docs-forge.

Claude Code

Copy the canonical skill folder into ~/.claude/skills/docs-forge and invoke /docs-forge.

Antigravity

Copy the adapter AGENTS.md, with optional GEMINI.md overrides for Antigravity-native sessions.

Other Agents

Use the universal AGENTS.md playbook for tools that read repository instructions.

Operating model

Seven phases, saved on disk.

  1. Scope

    Confirm audience, docs type, framework, output path, and existing docs.

  2. Ingest

    Build .docs-forge/kb/ from the entire codebase, with deeper passes over metadata, routes, exports, infra, and tests.

  3. Gap-analyse

    Write open questions before asking the user anything.

  4. Elicit

    Batch blocking questions and store answers for later sessions.

  5. Generate

    Create framework-native pages with citations back to source files.

  6. Capture

    Optionally run the project to collect screenshots or walkthrough media.

  7. Assemble

    Build navigation, validate links, and produce a completeness review.

Install

Install for the agent you use.

The official public Codex Plugin Directory is not self-serve yet, so Docs Forge is distributed through Git-backed installation paths and portable adapter files.

Codex codex plugin marketplace add adhit-r/docs-forge --ref v0.2.0
Claude Code git clone --branch v0.2.0 --depth 1 https://github.com/adhit-r/docs-forge.git
mkdir -p ~/.claude/skills
cp -R docs-forge/plugins/docs-forge/skills/docs-forge ~/.claude/skills/docs-forge
Antigravity cp docs-forge/adapters/antigravity/AGENTS.md ./AGENTS.md
